学习过程中遇到的各种问题解决

发表于2020-04-20更新于2020-05-12字数统计801阅读时长6分阅读次数

本篇博客的用意

在学习过程中，我们肯定会遇到各种各样的问题，有的可能会自动消失，有的会让我们焦头烂额
此篇博客收集了我最近遇到的一些问题和解决的办法，可能并不适用每个人，但也具有一定的代表性
问题主要是关于Android、大数据处理、虚拟机、IntelliJ IDEA

不定期更新(2020.5.12更新)

  • 之前运维和大数据主虚拟机都是用的一个虚拟机，最近大数据配置Hive需要MySQL，由于之前运维安装的是Mysql8.0.11，大数据需要Mysql5.x
  • 搞了半天一大堆问题，第二天才发现是因为需要的Mysql版本不同，导致步步都有问题，所以大部分都是关于Linux下Mysql的问题

Android

  • Android Studio 启动项目出现unsupported modules detected

  • Static interface methods are only supported starting with Android N (–min-api 24): void butterknife

  • Android studio配置Flutter教程，解决creating flutter project长时间卡停

  • Flutter报错之Waiting for another flutter command to release the startup lock解决方案

  • Android 9.0使用WebView加载Url时，显示页面无法加载

  • void android.widget.TextView.setText(java.lang.CharSequence)’ on a null object reference,后来发现绑定的id写错了，导致传的空值

Mysql

  • Linux安装mysql-8.0.12-linux-glibc2.12-x86_64.tar.xz

  • CentOS7中mysql-8.0.13-linux-glibc2.12-x86_64的安装

  • mysql 5.7.12—-bin/mysqld –initialize –user=mysql出错

  • mysql报错Can’t connect to local MySQL server through socket ‘/usr/local/mysql/mysql/mysql.sock’ (2)

  • mysql在linux下的my.cnf文件在哪里！

  • 解决Centos7.4无法启动mysql异常

  • 重启linux系统后 启动mysql ERROR! The server quit without updating PID file /var/run/mysqld/mysqld.pid

  • mysqld_safe Directory ‘/var/lib/mysql’ for UNIX socket file don’t exists.

  • mysql-创建用户报错ERROR 1396 (HY000): Operation CREATE USER failed for ‘XXXX’@’XXXX’

  • 新版SQL授权用户时报错 near ‘IDENTIFIED BY ‘密码’ with grant option’ at line 1

  • 解决ERROR 2002 (HY000): Can’t connect to local MySQL server through socket ‘/tmp/mysql.sock’ (2)

  • JDBC连接MySQL报错Unknown system variable ‘query_cache_size’

  • Unable to load authentication plugin ‘caching_sha2_password’.错误

  • CENTOS下如何卸载用RPM安装的MYSQL
